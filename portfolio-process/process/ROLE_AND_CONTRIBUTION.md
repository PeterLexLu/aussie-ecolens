# 产品经理与 Developer D 职责说明

## 项目定位

- **客户：** 澳大利亚原生动物保护公益组织。
- **交付类型：** 多云基础设施、后端能力与机器学习推理集成外包。
- **项目周期：** 需求与规划 2 周、开发 6 周、测试与客户 Demo 2 周、交付 1 周，6 月底完成交接后进入售后维护。
- **UI 边界：** 仓库 UI 仅用于 Demo 和功能测试，客户最终 UI 由独立外包团队负责。

## 个人角色

**Product Manager / Developer D**

这是一个双重角色：作为产品经理负责客户需求、PRD、Backlog、计划、验收和交付协调；作为 Developer D 参与测试工作流、接口集成验证及 Demo 支撑，但不将测试 UI 描述为最终产品 UI 设计。

## 产品经理职责

- 第一周直接与客户沟通，收集业务需求、使用场景和交付约束。
- 将客户需求转化为系统能力、非功能需求和验收条件。
- 组织团队可行性会议，对齐 AWS、GCP、媒体处理与安全方案。
- 第一周完成 PRD。
- 第二周拆分 30 个 User Stories，并制定甘特图、依赖和里程碑。
- 开发阶段主持每周一和周五 Stand-up Meeting。
- 周一安排当周 Stories，周五结算完成状态、风险和 Carry-over。
- 两周测试期内协调内部测试、客户 Demo、反馈分类和改进范围。
- 交付周协调客户及其 UI 外包团队完成 API、认证、状态和权限交接。
- 6 月底交付后参与代码售后、缺陷响应和按需改进。

## Developer D 职责

- 参与 Cognito 登录和受保护 API 的集成验证。
- 使用测试 UI 验证上传、媒体列表、查询、标签、删除和通知工作流。
- 验证 My Media、Shared Media 和 Results 的数据与权限表现。
- 支持 species、tag-count、file 和 thumbnail 查询的浏览器端测试。
- 配合 AWS、GCP 和媒体处理负责人定位跨云接口问题。
- 为客户 Demo 准备可重复执行的业务路径和测试数据。
- 在交付阶段向外部 UI 团队说明 API 行为、异步状态和错误处理。

## 团队协作

| 协作对象 | 对齐内容 |
| --- | --- |
| 客户 | 业务需求、优先级、Demo 反馈、交付范围和售后问题 |
| AWS 负责人 | Cognito、API Gateway、Lambda、S3、DynamoDB、SQS、SNS 与权限 |

## 作品集案例叙事

### Problem

客户需要可被正式 UI 集成的野生动物媒体后端能力，包括跨云推理、搜索、权限与通知，但多服务依赖、视频处理和跨团队交接提高了交付风险。

### Action

- 从客户访谈中提炼需求，并在第一周完成 PRD。
- 把范围拆成 30 个 User Stories 和甘特图。
- 用周一计划、周五结算的节奏管理六周开发。
- 协调 AWS、GCP 和媒体处理接口，使用测试 UI 做端到端验证。
- 在客户 Demo 后管理反馈和改进范围。
- 与客户 UI 供应商交接 API、权限、状态和错误行为。

### Result

- 完成认证、上传、异步推理、媒体查询、标签、删除和通知的多云能力闭环。
- 建立后端可信身份、所有权校验和受保护媒体访问。
- 通过 SQS、checksum 和视频抽帧聚合改善可靠性与资源使用。
- 6 月底完成客户与 UI 供应商交接，并持续提供代码售后支持。

### Reflection

后续可以进一步加强真实生产流量指标、可观察性 Dashboard、SLA/SLO 以及客户 UI 接入后的用户行为验证。

## 简历量化说明

“首屏时间下降约 70%”“操作时间减少约 90%”等数字只有在保留基线版本、测试环境、样本和测量方法时才使用。证据不足时，建议改写为具体机制或“显著降低”，避免在面试中无法解释口径。
