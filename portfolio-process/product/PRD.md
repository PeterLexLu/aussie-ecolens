# Aussie EcoLens MVP PRD

> 文档性质：根据客户需求、现有系统功能、项目报告与代码仓库补录。客户为澳大利亚原生动物保护公益组织。本团队提供多云基础设施与后端能力，不负责最终 UI；仓库中的 UI 仅用于 Demo 和功能测试。

## 1. 产品背景

客户需要一套可供其正式 UI 集成的多云野生动物媒体能力。野生动物图片与视频的人工整理、标记和检索成本较高，因此系统通过无服务器架构提供上传、AI 物种识别、标签检索、权限管理和通知等后端工作流。

## 2. 产品目标

在交付阶段验证一条可由客户 UI 调用的完整能力链路：

`登录 → 上传媒体 → 异步识别 → 查看结果 → 搜索与管理 → 订阅通知`

产品目标：

- 降低野生动物媒体整理和检索成本。
- 让用户能够理解媒体当前的处理状态。
- 使用自动标签与手动标签共同改善媒体可发现性。
- 在共享可见的前提下，保证修改与删除操作受所有权保护。
- 通过物种订阅让用户及时获知匹配的新媒体。

## 3. 客户与最终用户

### 野生动物研究者

需要集中保存媒体、快速检索物种记录，并修正识别标签。

### 环保组织成员

需要查看共享记录、关注指定物种，同时避免他人误删或篡改数据。

### 自然摄影爱好者

需要上传照片或视频，自动识别动物并整理个人媒体。

## 4. 交付边界

- 本团队负责 AWS/GCP 基础设施、后端 API、媒体处理、机器学习推理集成、安全权限、测试与交接。
- 本团队编写的 UI 用于 Demo、浏览器功能测试和端到端验证。
- 客户最终 UI 由独立外包团队负责，本团队在交付阶段提供 API 和集成支持。
- 测试 UI 的视觉设计、组件和交互细节不应被描述为最终产品设计成果。

## 5. MVP 范围

### P0：核心能力

- Cognito 用户认证与受保护访问。
- 图片和视频上传。
- S3 媒体存储与 DynamoDB 元数据存储。
- SQS 异步处理和 Cloud Run 模型推理。
- 自动标签、缩略图和视频标签聚合。
- My Media 与 Shared Media 展示。
- 物种标签、文件和缩略图查询。
- 手动标签管理与所有权校验。
- 所有者删除媒体。
- 物种订阅与 SNS 邮件通知。
- 基于 checksum 的重复上传检测。

### P1：体验完善

- 清晰的加载、空状态、成功和错误反馈。
- 上传与处理中状态反馈。
- 账户菜单、批量标签操作和媒体预览。
- 桌面浏览器可用性验证。

### 暂不纳入

- 原生移动端应用。
- 在线模型训练和实时视频识别。
- 复杂组织权限、社交互动和付费能力。
- 面向运营人员的完整数据分析后台。

## 7. 非功能需求

### 安全

- 后端从 Cognito JWT 推导用户身份，不信任前端提交的 owner 字段。
- 原始媒体和缩略图通过受保护接口访问。
- 删除和手动标签修改必须在后端校验所有权。
- 密钥、Token、`.env`、本地数据库和媒体文件不得提交到版本库。

### 可靠性

- 上传与推理解耦，失败任务可通过队列重试。
- 重复消息处理不得产生重复媒体记录。
- 推理失败应保留可诊断状态，不能静默失败。

### 性能

- 图片与视频使用异步处理，避免长耗时请求阻塞前端。
- 视频按固定频率抽帧并聚合结果，控制处理成本。
- 模型服务与应用服务独立扩缩容。

## 8. 成功指标

以下指标作为复盘与作品集说明的建议口径，只有在具备测试记录时才填写实际数值：

- 核心任务完成率：成功完成登录、上传、识别和检索的测试比例。
- 端到端成功率：测试环境中完整流程的成功次数占比。
- 重复上传拦截率：已知重复样本被正确识别的比例。
- 权限测试通过率：跨用户修改、删除与访问用例的通过比例。
- 图片与视频平均处理时长。
- P0 缺陷数量和发布阻塞缺陷关闭率。

## 9. 验收出口

- P0 用户故事全部通过验收。
- 图片和视频的端到端流程在测试环境可重复运行。
- 未认证访问和越权修改测试通过。
- 失败场景有用户可理解的反馈或可查询状态。
- 部署、测试和环境配置说明可供其他成员复现。
