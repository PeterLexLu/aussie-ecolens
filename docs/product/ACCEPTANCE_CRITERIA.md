# 用户故事与验收标准

> 本文档根据现有实现补录，用于说明需求如何转化为可测试结果。

## US-01 用户认证

**Given** 用户未登录，**When** 访问受保护功能，**Then** 系统拒绝请求并引导登录。

**Given** 用户携带有效 Cognito Token，**When** 调用受保护 API，**Then** 后端从 Token 获取用户身份。

**Given** 用户修改前端请求中的 owner 值，**When** 请求删除或修改媒体，**Then** 后端仍以 Token 身份进行授权判断。

## US-02 媒体上传

**Given** 已登录用户选择受支持的图片或视频，**When** 上传成功，**Then** 媒体进入存储并生成对应元数据记录。

**Given** 上传任务进入处理队列，**When** 推理尚未完成，**Then** Demo/测试客户端不会把它展示为已完成结果，且接口返回可供正式 UI 使用的处理状态。

**Given** 用户上传与平台已有文件 checksum 相同的文件，**When** 后端完成校验，**Then** 不重复执行存储和推理工作。

## US-03 AI 识别

**Given** 图片处理成功，**When** 用户查看媒体，**Then** 可以看到缩略图和模型识别标签。

**Given** 视频处理成功，**When** 用户查看结果，**Then** 系统展示按时间采样并聚合后的标签结果。

**Given** Cloud Run 调用失败，**When** 队列重试仍未成功，**Then** 日志中可以依据媒体标识定位失败阶段。

## US-04 媒体检索

**Given** 数据库存在匹配物种标签的媒体，**When** 用户执行标签查询，**Then** 返回符合条件的结果。

**Given** 没有匹配结果，**When** 查询完成，**Then** API 返回可区分的空结果，Demo/测试客户端不把它显示为系统错误。

**Given** 用户提交文件或缩略图查询，**When** 请求有效，**Then** 查询结果显示在独立结果区域，不污染媒体库状态。

## US-05 手动标签

**Given** 当前用户是媒体所有者，**When** 添加或删除手动标签，**Then** 数据库和界面状态一致更新。

**Given** 当前用户不是媒体所有者，**When** 尝试修改标签，**Then** 后端拒绝操作。

## US-06 删除媒体

**Given** 当前用户是媒体所有者，**When** 确认删除，**Then** 相关媒体记录按照系统删除策略处理。

**Given** 当前用户不是媒体所有者，**When** 发出删除请求，**Then** 后端返回无权限结果且数据保持不变。

## US-07 订阅通知

**Given** 用户创建物种订阅并完成邮箱确认，**When** 新媒体包含匹配标签，**Then** SNS 发送通知。

**Given** 用户取消订阅，**When** 后续出现匹配媒体，**Then** 该订阅不再触发通知。

## 发布级验收清单

- [ ] 登录、退出和 Token 失效流程通过。
- [ ] 图片上传、识别与显示通过。
- [ ] 视频上传、抽帧、聚合与显示通过。
- [ ] 重复上传检测通过。
- [ ] 标签、文件和缩略图查询通过。
- [ ] 所有者与非所有者权限用例通过。
- [ ] 订阅创建、确认、触发与取消通过。
- [ ] API 的加载、空结果和错误状态可被客户端区分，Demo/测试客户端能够验证这些状态。
- [ ] 验收不将 Demo/测试客户端视为客户最终 UI。
- [ ] 仓库中不存在真实密钥、本地数据库和用户媒体。
